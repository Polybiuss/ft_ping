-p PATTERN / --pattern=PATTERN :
    valeur hexadecimale qui va remplir le packet de donnees arbitraire
    envoyer avec l'ICMP ECHO en hexa!!! 
    le pattern est taille de 1 octets a 16 octets
    si la taille excede les 16 octets on ne prends que les 16 premiers octets
    le remplissage doit se faire octets par octets!!
    ff = 1 octet | 16 x 16 = 256
    en hexa une taille de deux equivaut a 1 octet la longueur qu'on donne
    a -p determine la taille en octet du pattern:
    ex : ff = 1 octet
         0000 = 2 octet
    plusieurs choix de syntaxes possible:
    -p PATTERN / -pPATTERN
    --pattern=PATTERN / --pattern PATTERN
-r / --ignore-routing :
    ignore la table de routage cherche a aller directement a l'addresse IP en local
    le resultat sera determiner avec la fonction sendto qui retournera -1
    et settera la valeur de errno
    pour ignorer la table de routage utilise : setsockopt avec option SO_DONTROUTE
-s N / --size=NUMBER :
    determine la taille des donnes arbitraire envoyer avec l'en-tete ip et icmp
    point important:
    un datagramme ipv4 a pour taille max 65535 octets,
    en deduisant la taille d'un en-tete ip sans options : 20 octet
    et la taille d'un entete icmp : 8 octet
    il reste 65507 octet pour la taille du packet mais c'est une erreur
    car on ne sait pas si le icmp echo reply aura un en tete ip sans options
    si il en possede on overflow notre ipv4 il faut prendre en compte
    les valeur max de l'en-tete ip avec options pour determiner
    la size max des donnes arbitraire
    taille max de header ip : 60 octets
    plusieurs choix de syntaxe possible :
    -sNUMBER / -s NUMBER
    --size=NUMBER / --size NUMBER
-T NUMBER / --tos=NUMBER:
    definie le type of service , gere les priorite du ipv4 dans le passage des table de routage jusqu'a l'ip
    valeur comprise entre 0 et 255
    plusieurs choix possible de syntaxe:
    -TNUMBER / T NUMBER
    --tos=NUMBER / --tos NUMBER
